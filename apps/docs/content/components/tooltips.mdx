Tooltips provide auxiliary information regarding the functionality or state of a UI element. Bug On MD3 Expressive tooltips support two variants: **Plain** for simple labels and **Rich** for detailed content that can include interactive actions.

<ComponentPreview name="tooltip-basic" />

## Plain Tooltips

Plain tooltips provide succinct descriptions for interface elements. They are best utilized for labeling icons or providing brief contextual information that does not require user interaction.

- **Placement**: Automatically toggles between top, bottom, left, and right based on available viewport space.
- **Activation**: Responds to `hover`, `focus`, and `long-press` (for mobile/touch devices).
- **Global Exclusivity**: Only one tooltip can be active at a time across the entire application.

## Rich Tooltips

Rich tooltips offer more detailed information and can include a title, body text, and up to two interactive buttons.

<ComponentPreview name="tooltip-rich" />

- **Actions**: Supports interactive elements such as buttons for further navigation or confirmation.
- **Persistence**: Rich tooltips can be configured to remain visible until explicitly dismissed or after a custom duration.
- **Caret**: An optional directional arrow (caret) can be enabled to point specifically at the anchor element.

## Triggers

Tooltips support various trigger types to accommodate different usage contexts:

| Trigger | Behavior |
|---------|---------|
| `hover` | Displays after a delay (default 400ms) on mouse entry. Remains visible when transitioning from the anchor to the tooltip content. |
| `focus` | Displays immediately when the element receives keyboard focus. |
| `long-press` | Displays after a sustained press (500ms) on touch screens. |
| `click` | Displays upon click (commonly used for Rich Tooltips requiring interaction). |

> [!TIP]
> **Hover Intent**: When utilizing the `hover` trigger, the tooltip does not dismiss immediately upon mouse exit from the anchor. It waits for a specific duration (`hideDelay`) to allow the user to move the cursor into Rich Tooltips containing interactive action buttons.

## Usage

```tsx
import { 
  Button, 
  TooltipBox, 
  PlainTooltip, 
  RichTooltip 
} from "@bug-on/md3-react";

// Basic Plain Tooltip
<TooltipBox tooltip={<PlainTooltip>Save to cloud</PlainTooltip>}>
  <Button icon={<Icon name="cloud_upload" />} />
</TooltipBox>

// Rich Tooltip with Action
<TooltipBox
  tooltip={
    <RichTooltip
      title="Permissions Required"
      action={<Button colorStyle="text">Settings</Button>}
    >
      This action requires location access to function correctly.
    </RichTooltip>
  }
>
  <Button>Enable Location</Button>
</TooltipBox>
```

## Accessibility

- **ARIA Role**: Tooltips utilize `role="tooltip"` and are automatically linked to the anchor element via `aria-describedby`.
- **Keyboard Navigation**: Tooltips appear on `focus` and can be dismissed using the `Escape` key. If a `click` trigger is used, users can activate it via `Enter` or `Space`.
- **Touch Support**: On mobile devices, tooltips respond to `long-press` interactions to avoid conflicts with scroll gestures.
- **Reduced Motion**: Adheres to the system-level `prefers-reduced-motion` setting, transitioning from spring-based scaling to a simple fade effect.

## API Reference

### TooltipBox

The wrapper component governing positioning, portal rendering, and interaction triggers.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `tooltip` | `ReactNode` | — | Tooltip content (typically `PlainTooltip` or `RichTooltip`). |
| `placement` | `TooltipPlacement` | `"top"` | Preferred positioning: `"top" | "bottom" | "left" | "right" | "auto"`. |
| `trigger` | `TooltipTrigger[]` | `["hover", "focus"]` | Array of triggers: `"hover" | "focus" | "long-press" | "click"`. |
| `spacingFromAnchor` | `number` | `4` | Spacing (in pixels) between the anchor and the tooltip. |
| `disabled` | `boolean` | `false` | If true, the tooltip will not be displayed. |
| `showDelay` | `number` | `400` | Delay (ms) before the tooltip appears on hover. |
| `hideDelay` | `number` | `200` | Delay (ms) before the tooltip hides after the mouse exits the interaction zone. |

### PlainTooltip

|------|------|---------|-------------|
| `caret` | `CaretConfig` | `{ enabled: false }` | Configuration for the optional directional caret. |
| `containerColor` | `string` | — | Background color override. |
| `textColor` | `string` | — | Text color override. |

### RichTooltip

Content-rich variant with title and interactive buttons.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `ReactNode` | — | Optional header title. |
| `action` | `ReactNode` | — | Optional interactive buttons. |
| `caret` | `CaretConfig` | — | Configuration for the optional directional caret. |
| `colors` | `RichTooltipColors` | — | Color overrides for individual elements (container, title, body, action). |
