## Search

The Search component allows users to input search queries and view suggestions or results. Following the Material Design 3 Expressive guidelines, this component supports flexible display modes—collapsing into a pill-shaped **SearchBar** and expanding into a **SearchView** overlay with smooth Framer Motion layout transitions.

<ComponentPreview name="search-basic" />

### Overview

Our Search system is highly modular and WAI-ARIA compliant out of the box. It manages focus, keyboard navigation (Arrow keys, Enter, Escape), and semantic HTML elements natively.

- **SearchBar**: The collapsed state. Click or focus to transition to the active mode.
- **SearchViewDocked**: An overlay that typically hovers just below the app bar. Suitable for large screens (Tablet, Desktop).
- **SearchViewFullScreen**: An overlay that completely masks the screen underneath using a portal. Common on Mobile platforms.

## Variants

### Docked, Contained Style (Default)

The standard search behavior. The view remains anchored to the input area and shares the background container.

### Docked, Divided Style

The `divided` style variant places a subtle horizontal ruler line beneath the search header, separating the input field from the drop-down results. Useful for search views with heavy content. 

<ComponentPreview name="search-docked-divided" />

### Docked with Gap

When `hasGap=true` is provided, the docked view detaches slightly (2dp space) from the header section.

<ComponentPreview name="search-docked-gap" />

### FullScreen Viewer

If `variant="fullscreen"` is used, the search expands dramatically into a full screen dialog, rendered cleanly into a React Portal to avoid `z-index` collisions.

## API Reference

### Search API

The main orchestrator component managing the transition between the SearchBar and the active SearchView.

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `query` | `string` | - | **Required.** Controlled search query text. |
| `onQueryChange` | `(query: string) => void` | - | **Required.** Callback triggered to change query text. |
| `onSearch` | `(query: string) => void` | - | Callback triggered when the user commits to a search (Presses Enter or submits the form). |
| `active` | `boolean` | `false` | Specifies whether the SearchView is open. |
| `onActiveChange` | `(active: boolean) => void` | - | Callback triggered when the component requests to open/close. |
| `variant` | `docked` \| `fullscreen` | `docked` | Visual layout variant of the active view. |
| `styleType` | `contained` \| `divided` | `contained` | Whether a divider separates the input from the results. |
| `textAlign` | `left` \| `center` \| `right` | `left` | Alignment of the placeholder when inactive. Smoothly slides to `left` when focus/active. |
| `hasGap` | `boolean` | `false` | (Docked only) Configures a 2dp gap between header and body. |
| `leadingIcon` | `ReactNode` | `<SearchIcon />` \| `<ArrowBackIcon />` | The leading action graphic depending on view state. |
| `trailingIcon` | `ReactNode` | - | The trailing graphic when query is empty. Replaced by "Clear" button dynamically. |
| `searchId` | `string` | `(auto-generated)` | Links the Framer Motion `layoutId` and the input `id`. |
| `listboxId` | `string` | `(auto-generated)` | References the WAI-ARIA listbox id for `aria-controls`. |

### Accessibility

- Natively uses `<search>` HTML tags for the container.
- Input uses `role="combobox"` with correct `aria-expanded` and `aria-controls` bindings.
- Dynamic `aria-activedescendant` is applied as the user navigates results using Arrow keys.
- Search items inside `children` must manually apply `role="option"` and `tabIndex={-1}` for semantic validity.
