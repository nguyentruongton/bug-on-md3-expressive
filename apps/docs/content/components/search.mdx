The Search component allows users to input search queries and view suggestions or results. Following the Material Design 3 Expressive guidelines, this component supports flexible display modes—collapsing into a pill-shaped **SearchBar** and expanding into a **SearchView** overlay with smooth Framer Motion layout transitions.

## Introduction

The MD3 Expressive Search system is a highly modular set of components designed for both simple queries and complex filtering. It seamlessly transitions between a compact, non-intrusive pill (SearchBar) and a focused results view (SearchView). Using Framer Motion's shared element transitions, the expansion feels fluid and natural, anchoring the user's focus.

## Anatomy

- **Search Bar (Collapsed)**: A pill-shaped input field that serves as the entry point.
- **Search View (Active)**: The expanded panel that displays suggestions and results.
- **Leading Icon**: Typically a "Search" icon when collapsed, and a "Back" arrow when active.
- **Input Field**: The area for typing the query.
- **Trailing Actions**: Icons for clearing the text or triggering secondary actions (e.g., Microphone).
- **Results / Suggestions List**: The scrollable area below the input.

## Variants

### Docked View

The search view expands directly below the input field, keeping the rest of the screen visible. Ideal for desktop and tablet layouts.

- **Contained**: The input and results share a single container.
- **Divided**: A horizontal line separates the input from the results.
- **Gap**: The results panel is detached from the input header.

### Full-screen View

The search view expands to cover the entire viewport. Best for mobile devices to provide maximum focus on results.

## Features

### Shared Element Transitions

When the user focuses the SearchBar, it morphs into the SearchView header using a `layoutId` transition, ensuring a consistent visual anchor.

### Text Alignment

The placeholder text can be centered in the collapsed state and smoothly slides to the left as the field is focused, matching MD3 expressive motion.

## Usage

### Basic Usage

```tsx
import { Search } from "@bug-on/md3-react";

export function SearchExample() {
  const [query, setQuery] = useState("");
  const [active, setActive] = useState(false);

  return (
    <Search
      query={query}
      onQueryChange={setQuery}
      active={active}
      onActiveChange={setActive}
      placeholder="Search photos..."
    >
      <div className="p-4">
        {/* Render your search results or suggestions here */}
        <p>Suggestions go here...</p>
      </div>
    </Search>
  );
}
```

## Best Practices

### Dos
- [ ] Use **Docked** search for desktop layouts and **Full-screen** for mobile.
- [ ] Provide immediate visual feedback as the user types (suggestions).
- [ ] Include a "Clear" button in the trailing icon slot to help users reset their query.
- [ ] Ensure the placeholder text is descriptive of what can be searched.

### Don'ts
- [ ] Don't hide the search entry point in deeply nested menus if search is a primary app function.
- [ ] Avoid long delays in displaying search results; use a loading state if necessary.
- [ ] Don't use search for filtering a list that has fewer than 10 items.

## Accessibility

- **Roles**: Correctly implements `role="search"` and `role="combobox"`.
- **States**: Uses `aria-expanded` and `aria-haspopup` to communicate the state of the results view.
- **Keyboard**: Navigate results with Arrow keys. `Enter` to select, `Esc` to close.
- **Focus**: Focus is automatically managed during the transition between states.
