Snackbars inform users of a process that an app has performed or will perform. They appear temporarily, towards the bottom of the screen. They shouldn't interrupt the user experience, and they don't require user input to disappear.

<ComponentPreview name="snackbar-basic" />

## Introduction

The MD3 Expressive Snackbar is a lightweight notification component used for low-priority feedback. It provides a non-disruptive way to confirm actions (e.g., "Message sent") or inform about background processes. Built with a queuing system, it ensures that messages are delivered sequentially without overlapping.

## Anatomy

- **Container**: A dark surface with rounded corners (4dp) that floats at the bottom of the screen.
- **Message Text**: A short string describing the event.
- **Action (Optional)**: A single text button for the user to respond to the notification (e.g., "Undo").
- **Close Button (Optional)**: A dedicated "X" button to dismiss the snackbar immediately.

## Features

### Queuing System

If multiple snackbars are triggered in rapid succession, the system automatically queues them, showing one at a time to avoid overwhelming the user.

### Automatic Dismissal

Snackbars are temporary and dismiss themselves after a delay (default 4s for short, 7s for long).

## Usage

### Setup

Enable snackbars in your root layout via `MD3ThemeProvider`:

```tsx
import { MD3ThemeProvider } from "@bug-on/md3-react";

export function Root({ children }) {
  return (
    <MD3ThemeProvider enableSnackbar>
      {children}
    </MD3ThemeProvider>
  );
}
```

### Triggering a Snackbar

Use the `useSnackbar` hook to access the `showSnackbar` function.

```tsx
import { useSnackbar } from "@bug-on/md3-react";

export function MyComponent() {
  const { showSnackbar } = useSnackbar();

  const handleSave = async () => {
    // Logic...
    const result = await showSnackbar({
      message: "Profile updated",
      actionLabel: "Undo"
    });

    if (result === 'action-performed') {
      // Undo logic...
    }
  };

  return <Button onClick={handleSave}>Save</Button>;
}
```

## Best Practices

### Dos
- [ ] Keep messages short and direct.
- [ ] Use snackbars for non-critical confirmation messages.
- [ ] Provide an "Undo" action for destructive operations if possible.
- [ ] Ensure the snackbar doesn't obscure important UI elements like FABs or Navigation Bars.

### Don'ts
- [ ] Don't use snackbars for critical errors that require immediate attention (use a Dialog instead).
- [ ] Don't put more than one action in a snackbar.
- [ ] Avoid using snackbars for long-lasting or permanent information.

## Accessibility

- **Roles**: Uses `role="status"` with `aria-live="polite"` to ensure screen readers announce the message without interrupting the current task.
- **Duration**: Provides enough time for users to read the message. The `long` duration is recommended for snackbars with actions.
- **Interaction**: The action button is reachable via keyboard.

## API Reference

### useSnackbar

Hook providing access to the snackbar system.

| Property | Type | Description |
|----------|------|-------------|
| `showSnackbar` | `(config: SnackbarConfig) => Promise<string>` | Triggers a snackbar and returns its result. |

### SnackbarConfig

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `message` | `string` | — | **Required.** Message text. |
| `actionLabel` | `string` | — | Label for the action button. |
| `withDismissAction` | `boolean` | `false` | Shows a close icon. |
| `actionOnNewLine` | `boolean` | `false` | Moves action to a new line. |
| `duration` | `"short" \| "long" \| number` | `"short"` | Display time. |
| `className` | `string` | — | Custom classes. |
