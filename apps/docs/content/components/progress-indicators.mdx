Progress indicators show the real-time status of a process. Features include determinate and indeterminate modes, circular indicators, and expressive wavy linear bars.

<ComponentPreview name="progress-playground" />

## Introduction

The MD3 Expressive Progress Indicator provides visual feedback for tasks that take more than a few seconds. It supports both **Linear** and **Circular** formats, and offers a unique **Wavy** design style that adds a layer of sophisticated motion to the standard progress bar. It handles both determinate (known progress) and indeterminate (unknown duration) states gracefully.

## Anatomy

- **Track**: The background line or circle representing the total progress.
- **Indicator**: The active part of the bar or circle representing completed progress.
- **Wavy Element (Optional)**: A dynamic SVG path that adds a wave effect to the indicator.

## Features

### Determinate vs Indeterminate

- **Determinate**: Use when the percentage of completion is known.
- **Indeterminate**: Use when the duration of the task is unknown.

### Wavy Motion

The `shape="wavy"` prop transforms the flat progress bar into an animated wave. This expressive style is perfect for high-end interfaces that want to stand out.

## Usage

### Basic Linear Progress

```tsx
import { ProgressIndicator } from "@bug-on/md3-react";

<ProgressIndicator value={60} />
```

### Indeterminate Circular Progress

```tsx
<ProgressIndicator variant="circular" />
```

### Expressive Wavy Progress

```tsx
<ProgressIndicator 
  shape="wavy" 
  value={45} 
  amplitude={6} 
  wavelength={30} 
/>
```

## Best Practices

### Dos
- [ ] Use Determinate progress whenever the duration can be calculated.
- [ ] Provide an `aria-label` that describes what is being loaded.
- [ ] Use Linear progress for long processes like file uploads or page loads.
- [ ] Use Circular progress for smaller, contextual loading (e.g., inside a card).

### Don'ts
- [ ] Don't use a progress indicator if the task takes less than 1 second.
- [ ] Don't switch between determinate and indeterminate states frequently; it can be confusing.
- [ ] Avoid using too many wavy indicators on a single page, as the motion can be distracting.

## Accessibility

- **Roles**: Automatically applies `role="progressbar"`.
- **States**: Manages `aria-valuenow`, `aria-valuemin`, and `aria-valuemax` for determinate states.
- **Indeterminate**: Removes `aria-valuenow` to signal unknown progress to screen readers.

## API Reference

### ProgressIndicator

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"linear" \| "circular"` | `"linear"` | Visual type. |
| `value` | `number` | — | Progress percentage (0-100). |
| `shape` | `"flat" \| "wavy"` | `"flat"` | Visual style of the indicator. |
| `trackShape` | `"flat" \| "wavy"` | `"flat"` | Visual style of the track. |
| `size` | `number` | `48` | Circular diameter. |
| `trackHeight` | `number` | `4` | Thickness. |
| `amplitude` | `number` | `4` | Wave height. |
| `wavelength` | `number` | `20` | Wave cycle length. |
| `waveSpeed` | `number` | `1` | Animation speed. |
| `className` | `string` | — | Custom classes. |
