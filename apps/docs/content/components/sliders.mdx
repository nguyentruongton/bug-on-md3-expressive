Sliders are ideal controls when users need to select from a continuous or discrete range of values. The MD3 Expressive edition stands out with its **4px pill handle** that **compresses to 2px** when pressed, a 6px gap between the track and handle, and asymmetric corner radii (outer=full, inner=2px).

## Introduction

The MD3 Expressive Slider is a high-fidelity input component designed for precision and tactile feedback. It features a unique "pill-shaped" thumb that responds to touch by compressing, mimicking the physical sensation of pressing a button. It supports various configurations, including centered tracks for neutral-point adjustments and range sliders for selecting intervals.

## Anatomy

- **Track**: The horizontal (or vertical) bar representing the range of values.
- **Thumb (Handle)**: The interactive element the user slides. Features a 4dp pill shape that compresses to 2dp on press.
- **Active Track**: The highlighted portion of the track indicating the selected value.
- **Inactive Track**: The remaining portion of the track.
- **Value Indicator (Optional)**: A tooltip that displays the current numerical value during interaction.
- **Ticks (Optional)**: Visual marks indicating discrete steps on the track.

## Variants

### Continuous vs Discrete

- **Continuous**: Allows selection of any value within the range.
- **Discrete**: Snaps to specific intervals defined by the `step` prop. Ticks can be enabled to visualize these steps.

### Centered Track

Ideal for settings that have a neutral starting point (e.g., exposure or balance), where the active track grows from the center outwards.

## Features

### Inset Icons

For larger track sizes (M, L, XL), you can place icons directly inside the track. These icons automatically transition between the active and inactive track segments.

<ComponentPreview name="slider-inset-icon" />

### Thumb Compression

On press, the thumb width reduces from 4dp to 2dp, providing a subtle but powerful tactile feedback loop.

### Orientation

Supports both horizontal and vertical layouts. Vertical sliders are useful for volume or intensity controls in dense interfaces.

## Usage

### Basic Usage

```tsx
import { Slider } from "@bug-on/md3-react";

const [value, setValue] = useState(50);

<Slider 
  value={value} 
  onValueChange={setValue} 
  min={0} 
  max={100} 
/>
```

### Range Selection

```tsx
import { RangeSlider } from "@bug-on/md3-react";

const [range, setRange] = useState([20, 80]);

<RangeSlider 
  value={range} 
  onValueChange={setRange} 
  min={0} 
  max={100} 
/>
```

## Best Practices

### Dos
- [ ] Use sliders for settings where the exact value is less important than the relative change (e.g., volume, brightness).
- [ ] Provide immediate feedback as the user slides the thumb.
- [ ] Use `isCentered` for adjustments with a logical zero or middle point.
- [ ] Ensure the slider has an `aria-label` or is linked to a visible label.

### Don'ts
- [ ] Don't use a slider for entering precise data where a Text Field would be more efficient.
- [ ] Don't crowd too many sliders into a single view.
- [ ] Avoid using Inset Icons for small track sizes (XS, S) as they become illegible.

## Accessibility

- **Keyboard**: Navigate with Arrow keys (fine-tuning) and Page Up/Down (large steps).
- **Roles**: Correctly implements `role="slider"`.
- **States**: Communicates current value via `aria-valuenow`.
- **Touch**: Provides a large 48x48dp touch target for the thumb.

## API Reference

### Slider

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `number` | — | Controlled value. |
| `onValueChange` | `(v: number) => void` | — | Value change callback. |
| `min` | `number` | `0` | Minimum value. |
| `max` | `number` | `100` | Maximum value. |
| `step` | `number` | `0` | Snap step (0 = continuous). |
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | Layout direction. |
| `trackSize` | `"xs" \| "s" \| "m" \| "l" \| "xl"` | `"m"` | Physical track thickness. |
| `isCentered` | `boolean` | `false` | Grows track from center. |
| `showValueIndicator`| `boolean` | `false` | Shows tooltip on interaction. |
| `insetIcon` | `ReactNode` | — | Icon inside the track. |

### RangeSlider

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `[number, number]` | — | Controlled range tuple. |
| `onValueChange` | `(v: [number, number]) => void` | — | Range change callback. |
