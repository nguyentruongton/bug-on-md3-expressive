Sliders are ideal controls when users need to select from a continuous or discrete range of values. The MD3 Expressive edition stands out with its **4px pill handle** that **compresses to 2px** when pressed, a 6px gap between the track and handle, and asymmetric corner radii (outer=full, inner=2px).

## Configurations

### Centered Track

When `isCentered={true}` is enabled, the active track segment grows from the center (50%) in either direction instead of from the `min` value. This is perfect for adjustments with a neutral zero point (e.g., audio balance, exposure, pitch).

<ComponentPreview name="slider-centered" />

### Discrete Sliders

When a `step > 0` is provided, the slider becomes discrete and snaps to values. You can enable `showTicks={true}` to display visual marks at each step. By default, ticks are hidden to keep the UI clean for wider ranges.

### Inset Icons

MD3 Expressive Slider supports **Inset Icons** — icons placed directly inside the track. These icons automatically transition between the active and inactive track segments based on the available space.

> [!IMPORTANT]
> **Design Constraint**: Inset icons are only available for track sizes **M, L, and XL** (height ≥ 12px). They should not be used with Centered or Range sliders.

<ComponentPreview name="slider-inset-icon" />

## Orientation

Set `orientation="vertical"` to rotate the slider. A vertical slider requires a container with a fixed height. The Y-axis is inverted: sliding up increases the value, mimicking physical volume faders.

<ComponentPreview name="slider-vertical" />

## Range Selection

`RangeSlider` displays two handles, with the active track segment positioned between them. Crossover prevention is built-in, ensuring the handles cannot pass each other.

## API Reference

### Slider

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `number` | — | Controlled current value. |
| `defaultValue` | `number` | `midpoint` | Initial value for uncontrolled usage. |
| `onValueChange` | `(v: number) => void` | — | Callback when value changes during interaction. |
| `onValueChangeEnd` | `(v: number) => void` | — | Callback when interaction finishes. |
| `min` | `number` | `0` | Minimum value. |
| `max` | `number` | `100` | Maximum value. |
| `step` | `number` | `0` | Snap step size. `0` = continuous. |
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | Layout direction. |
| `trackSize` | `"xs" \| "s" \| "m" \| "l" \| "xl"` | `"m"` | Physical size of the track. |
| `variant` | `"primary" \| "secondary" \| "tertiary" \| "error"` | `"primary"` | Color palette. |
| `trackShape` | `"md3" \| "full" \| number` | `"md3"` | Corner radius configuration. |
| `isCentered` | `boolean` | `false` | Grows active track from 50%. |
| `disabled` | `boolean` | `false` | Disables interaction. |
| `showValueIndicator` | `boolean` | `false` | Shows value tooltip on hover/drag. |
| `showTicks` | `boolean` | `false` | Shows stop marks (requires `step > 0`). |
| `insetIcon` | `ReactNode` | — | Icon inside leading track segment. |
| `insetIconAtMin` | `ReactNode` | — | Swap icon when value equals `min`. |
| `insetIconTrailing` | `ReactNode` | — | Icon inside trailing track segment. |
| `insetIconAtMax` | `ReactNode` | — | Swap icon when value equals `max`. |
| `formatValue` | `(v: number) => string` | `String(v)` | Custom tooltip value formatting. |
| `aria-label` | `string` | — | Screen reader label. |
| `aria-labelledby` | `string` | — | ID of a visible label element. |
| `className` | `string` | — | Custom wrapper CSS class. |

### RangeSlider

Inherits all props from `Slider` (except `isCentered`). The following props are overridden:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `[number, number]` | — | Controlled `[start, end]` tuple. |
| `defaultValue` | `[number, number]` | `[min, max]` | Initial tuple for uncontrolled usage. |
| `onValueChange` | `(v: [number, number]) => void` | — | Callback with new tuple. |
| `onValueChangeEnd` | `(v: [number, number]) => void` | — | Callback when interaction finishes. |

## Accessibility

- Thumbs use `role="slider"` with appropriate `aria-valuenow`, `aria-valuemin`, and `aria-valuemax`.
- Touch targets are at least **48x48px** to meet MD3 requirements.
- Full keyboard support: arrows (step), page up/down (10%), home/end (edges).
- Supports `prefers-reduced-motion` to disable animations.
- Focus rings use the `secondary` color per MD3 color specs.
