Tabs facilitate the organization of content into clear sections without requiring page navigation. Bug On MD3 Expressive Tabs support two variants: **primary** (indicator scales to content) and **secondary** (full-width indicator with a divider). Both variants support icons, horizontal scrolling, full keyboard navigation, and integrated **Badges**.

<ComponentPreview name="tabs-primary" />

## Secondary Tabs

Secondary Tabs feature a divider line at the base of the tab bar and an indicator that spans the full width of the tab. These are ideal for peer-level content or related filters. According to MD3 specifications, Secondary Tabs typically utilize text-only labels or **inline icons**.

<ComponentPreview name="tabs-secondary" />

## Tabs with Badges

You can attach a **Badge** to any Tab to communicate status or notification counts. 
- For **Primary Tabs** (stacked icons): The Badge is automatically positioned at the top-trailing corner of the icon.
- For **Inline Icons** or **Text-only labels**: The Badge is displayed immediately following the label with a standard 4dp offset.

<ComponentPreview name="tabs-badges" />

## Inline Icons

By default, if an icon is provided, the Tab displays it above the label (stacked), increasing the height to 64dp. Set the `inlineIcon={true}` prop to align the icon horizontally with the label, maintaining a compact 48dp height.

<ComponentPreview name="tabs-inline-icon" />

## Scrollable Tabs

When the number of tabs exceeds the container width, enable `scrollable={true}` to allow horizontal scrolling. The MD3 specification requires 52px of edge padding. The system automatically scrolls the selected tab into view.

## Disabled Tabs

Disabled tabs cannot be clicked or focused via keyboard. The roving tabindex system automatically skips these tabs during arrow key navigation.

## Usage

```tsx
import { Tabs, TabsList, Tab, TabsContent, Badge, Icon } from "@bug-on/md3-react";

// Primary with Badge and Stacked Icon
<Tabs defaultValue="mail">
  <TabsList variant="primary">
    <Tab value="mail" icon={<Icon name="mail" />} badge={<Badge>12</Badge>}>
      Mail
    </Tab>
    <Tab value="chat" icon={<Icon name="chat" />}>
      Chat
    </Tab>
  </TabsList>
  <TabsContent value="mail">Mail content...</TabsContent>
</Tabs>

// Secondary with Inline Icon
<Tabs autoActivate defaultValue="all">
  <TabsList variant="secondary">
    <Tab value="all">All</Tab>
    <Tab value="starred" icon={<Icon name="star" />} inlineIcon badge={<Badge />}>
      Starred
    </Tab>
  </TabsList>
  <TabsContent value="all">Messages content...</TabsContent>
</Tabs>
```

## Primary vs Secondary Usage

| | Primary | Secondary |
|---|---|---|
| **Indicator** | Matches content width (min 24dp) | Spans full tab width |
| **Indicator Shape** | Top rounded `3px 3px 0 0` | Top rounded `3px 3px 0 0` |
| **Indicator Height** | `3dp` | `2dp` |
| **Divider** | None | Feature a bottom divider line |
| **Active Text Color** | `primary` | `on-surface` |
| **Best For** | High-level navigation, categorical selection | Filters, sub-sections |

## Accessibility

- `<TabsList>` utilizes `role="tablist"` – mandated by WAI-ARIA.
- **Roving tabindex**: Automatically manages focus, skipping disabled tabs.
- **RTL Support**: Inverts arrow key directions automatically when `dir="rtl"` is detected.
- **Auto-activate**: Supports selection immediately upon focus via the `autoActivate` prop.

## API Reference

### Tabs

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | — | Controlled active tab value. |
| `defaultValue` | `string` | — | Initial value for uncontrolled mode. Defaults to the first non-disabled tab. |
| `onValueChange` | `(value: string) => void` | — | Callback fired when the active tab changes. |
| `autoActivate` | `boolean` | `false` | If enabled, the tab is selected immediately upon keyboard focus. |
| `className` | `string` | — | Custom CSS class. |

### TabsList

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"primary" \| "secondary"` | — | **Required**. Visual style of the tab bar. |
| `scrollable` | `boolean` | `false` | Enables horizontal scrolling with 52px edge padding. |
| `backgroundColor` | `string` | — | Background color of the tab bar. |
| `className` | `string` | — | Custom CSS class. |

### Tab

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | — | **Required**. Unique identifier for the tab. |
| `icon` | `ReactNode` | — | Icon displayed alongside the label. |
| `inlineIcon` | `boolean` | `false` | If enabled, aligns the icon horizontally with the text (maintains 48dp height). |
| `badge` | `ReactNode` | — | Badge displayed on the tab to communicate status/notifications. |
| `disabled` | `boolean` | `false` | Disables interaction and navigation. |
| `className` | `string` | — | Custom CSS class. |

### TabsContent

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | — | **Required**. Must match the `value` of the corresponding `<Tab>`. |
| `className` | `string` | — | Custom CSS class for the content panel. |
