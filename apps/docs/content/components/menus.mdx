## Basic Usage

Menus appear upon interaction with a button, action, or other control. They display a list of choices, one per line. The MD3 Expressive Menu automatically applies shape morphing to its items.

<ComponentPreview name="menu-basic-demo" />

## Selection States

Menus can support single-select (usually via check icons) and multi-select behaviors. By passing the `selected` prop to a `MenuItem`, it automatically morphs its shape and animates a checkmark.

<ComponentPreview name="menu-selection-demo" />

## Grouping & Dividers

Use `MenuGroup` and `MenuDivider` to logically separate items. The `MenuGroup` component orchestrates the shape morphing of its children, giving leading items rounded top corners, middle items slightly rounded corners, and trailing items rounded bottom corners.

<ComponentPreview name="menu-grouped-demo" />

## Icons & Shortcuts

Menu items can include a `leadingIcon` for quick recognition and `trailingText` to indicate keyboard shortcuts.

<ComponentPreview name="menu-icons-demo" />

## Supporting Text

You can provide `supportingText` to give users more context about a specific menu option.

## Submenus (Nested Menus)

Menus can contain nested submenus for complex navigation or deep options trees. Use the `SubMenu` component inside a `MenuContent`, and ensure you add the `hasOverflow` prop to the parent `MenuContent` so the submenu can escape the parent's container bounds.

<ComponentPreview name="menu-submenu-demo" />

## Vibrant Variant

The MD3 Expressive Menu supports a `vibrant` color variant built on the `tertiary-container` design token. This is intended for high-emphasis menus that need to immediately draw the user's attention.

<ComponentPreview name="menu-vibrant-demo" />

## Vertical Menus

MD3 Expressive defines two variants of menus: the popup **Menu** (baseline) and the **Vertical Menu** — an always-visible, statically rendered list with no trigger or popup animation. Vertical menus are ideal for sidebars, settings panels, and inline navigation where content should always be in view.

Both sub-variants share the same shape morphing on hover, `colorVariant` support, and `MenuItem` features (icons, selections, shortcuts). They differ only in how groups are separated.

### Vertical Menu with Gap

Groups are separated by a 2dp visual gap (`SegmentedMenuTokens.SegmentedGap`). This is the default `separatorStyle` and produces a clean, floating-card effect for each group.

<ComponentPreview name="vertical-menu-gap-demo" />

### Vertical Menu with Divider

Groups are separated by a full-width horizontal rule in the `outline-variant` color (`MenuDefaults.HorizontalDividerPadding`). Use this when a clearer visual boundary between groups is needed.

<ComponentPreview name="vertical-menu-divider-demo" />

### Vertical Menu with Labels

You can add a `label` to `VerticalMenuGroup` to provide context for the grouped actions. This uses the MD3 `label-small` typography.

<ComponentPreview name="vertical-menu-label-demo" />

### Vertical Menu Vibrant Variant

Vertical menus also support the `vibrant` color variant using the `colorVariant` prop.
