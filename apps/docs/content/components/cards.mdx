Cards are flexible containers that group related information and actions. They adapt their rendering element based on interactivity — static for display, `<button>` for actions, or `<a>` for navigation. Expressive cards animate elevation on hover and show Ripple effects on tap.

## Introduction

The MD3 Expressive Card is a fundamental unit of UI that helps organize content into digestible blocks. It supports three primary variants: **Elevated**, **Filled**, and **Outlined**. Beyond just layout, these cards are "smart" and automatically adapt their HTML semantics and interaction patterns based on whether they are static, clickable, or linkable.

## Anatomy

- **Container**: The main surface of the card. Supports different elevation levels and corner radii.
- **Content Area**: A flexible space for text, images, and other components.
- **Actions (Optional)**: Buttons or links typically placed at the bottom of the card.
- **State Layer**: Interactive cards (button/link) show a hover state and a ripple effect on click/tap.

## Variants

### Color Styles

- **Elevated**: High emphasis, uses a shadow to create depth. Best for standout content on plain backgrounds.
- **Filled**: Medium emphasis, uses a solid background color (Surface Container). Good for grouping related items.
- **Outlined**: Low emphasis, uses a border. Best for dense layouts where many cards are visible.

### Interactive Types

The `Card` component automatically selects the correct element:
- **Static**: Renders as `<div>`. Use when the card is purely for display.
- **Button**: Renders as `<button>` when `onClick` is provided. Features elevation morphing and MD3 ripple.
- **Link**: Renders as `<a>` when `href` is provided.

## Usage

### Basic Static Card

```tsx
import { Card } from "@bug-on/md3-react";

<Card variant="elevated" className="p-6">
  <h3>Project Title</h3>
  <p>Some descriptive text about the project.</p>
</Card>
```

### Interactive Button Card

```tsx
<Card 
  onClick={() => console.log("Card clicked")}
  variant="filled"
  className="p-6"
>
  <h3>Settings</h3>
  <p>Manage your account preferences.</p>
</Card>
```

### Card with Images

```tsx
<Card className="overflow-hidden">
  <img src="/image.jpg" alt="Description" className="w-full h-48 object-cover" />
  <div className="p-4">
    <h3>Card Title</h3>
    <p>Supporting text...</p>
  </div>
</Card>
```

## Best Practices

### Dos
- [ ] Use cards to group related information that belongs together.
- [ ] Maintain consistent padding and typography across all cards in a layout.
- [ ] Use Outlined cards for lists or grids with many items to avoid "shadow clutter".

### Don'ts
- [ ] Don't use too many Elevated cards on a single page, as it can make the UI feel "heavy".
- [ ] Don't make a card interactive if it only contains other interactive elements (e.g., buttons). This can lead to nested button issues.
- [ ] Avoid putting too much content in a single card; keep it focused on a single subject.

## Accessibility

- **Keyboard Support**: Interactive cards (button/link) are fully reachable via `Tab` and triggerable via `Space`/`Enter`.
- **Roles**: Automatically applies `role="button"` or `role="link"` as appropriate.
- **Disabled State**: Sets `aria-disabled="true"` and removes the card from the tab order.
- **Focus**: Visible focus rings are applied to interactive cards.

## API Reference

### Card

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"elevated" \| "filled" \| "outlined"` | `"elevated"` | Visual style of the card. |
| `onClick` | `() => void` | — | Makes the card a `<button>`. |
| `href` | `string` | — | Makes the card a link (`<a>`). |
| `target` | `string` | — | Link target (e.g., `_blank`). |
| `disabled` | `boolean` | `false` | Disables interaction and dims the card. |
| `asChild` | `boolean` | `false` | Renders as the child element while keeping styles. |
| `className` | `string` | — | Additional CSS classes. |
