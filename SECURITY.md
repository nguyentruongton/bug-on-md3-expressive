# Security Policy

## Supported Versions

We currently support the following versions of the library with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| 0.x.x   | :x:                |

## Reporting a Vulnerability

Security is a high priority for our project. If you discover a security vulnerability, we'd appreciate your help in disclosing it to us in a responsible manner.

Please **DO NOT** report security vulnerabilities via public GitHub issues.

Instead, please report them to our maintainers through GitHub's Security Advisories feature or by contacting the maintainers directly.

We will try to respond to your report within 48 hours and work with you to resolve it as quickly as possible.

When reporting a vulnerability, please include the following information and any other details you think might be relevant:
- The package(s) and version(s) affected.
- A detailed description of the vulnerability.
- Steps to reproduce the vulnerability (a minimal code example or repository is highly appreciated).
- Any potential impact or damage it could cause.

Thank you for helping us keep our library safe!
